# 19. 실제 서비스형 UI/UX 보완 보고서

작성일: 2026-06-30

## 1. 작업 목표

BeautyLens JSP 시연 사이트가 개발/관리자 데모처럼 보이는 부분을 줄이고, 일반 사용자가 실제 화장품 추천 서비스처럼 자연스럽게 사용할 수 있도록 UX를 보완했다.

## 2. 메인 CTA 정리

- 제거: 큰 `로그인`, `관리자 페이지` CTA
- 유지/변경: `추천 받기`, `피부 타입 선택하기`
- 메인 설명 카드 문구를 사용자 중심으로 변경
  - 피부 타입별 추천
  - 리뷰 신호 분석
  - 사용자 의견 확인

## 3. 관리자 메뉴 노출 정책

상단 네비게이션 정책:

| 상태 | 표시 메뉴 |
|---|---|
| 미로그인 | 홈, 추천, 로그인 |
| USER | 홈, 추천, 마이페이지, 로그아웃 |
| ADMIN | 홈, 추천, 마이페이지, 관리자, 로그아웃 |

관리자 데이터 접근은 기존 관리자 API 권한 정책을 유지한다.

## 4. 추천 페이지 피부 타입 즉시 선택

추가 버튼:
- 건성
- 지성
- 복합성
- 민감성
- 중성

동작:
- 선택 즉시 해당 피부 타입 기준 추천을 다시 불러온다.
- 마이페이지의 회원 정보는 자동 변경하지 않는다.
- 선택된 피부 타입은 active 스타일과 현재 추천 기준 문구로 표시한다.

추가 API:

```text
GET /api/recommendations?skinType={skinType}&size=20
```

기존 `/api/recommendations/me`는 그대로 유지했다.

## 5. 마이페이지 내 댓글 목록

추가 섹션:

```text
내가 남긴 의견
```

표시 항목:
- 상품 이미지
- 브랜드
- 상품명 또는 displayName
- 댓글 내용
- 작성일
- 상태
- 상품 보러가기
- ACTIVE 댓글 삭제

추가 API:

```text
GET /api/members/me/comments
```

권한:
- 로그인 필요
- 미로그인 401

## 6. 상품 카드 및 필터 개선

- 상품 이미지 영역 비율 유지
- 상품명 2줄 제한
- 추천 이유 3줄 제한
- 버튼 하단 정렬
- 플랫폼/이미지 필터 active 상태 강화
- 현재 추천 기준과 필터 상태 텍스트 표시

## 7. 시연용 문구 제거

화면에서 제거한 유형:
- 시연 계정 노출
- 관리자 페이지 CTA
- Spring MVC Demo
- JSP + Vanilla JavaScript
- soft delete 같은 내부 구현 문구
- 내부 API 설명

시연 계정은 문서에만 유지한다.
