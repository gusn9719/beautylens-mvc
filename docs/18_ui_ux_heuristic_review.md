# 18. UI/UX 휴리스틱 리뷰 및 보완 결과

작성일: 2026-06-30

## 개요

BeautyLens JSP 시연 사이트를 제이콥 닐슨의 10가지 사용성 휴리스틱 기준으로 점검했다. 이번 보완은 새 기능 추가가 아니라 시연을 방해하는 UX/UI 문제를 최소 수정으로 개선하는 데 한정했다.

## 1. 시스템 상태의 가시성

- 현재 문제: 추천 페이지 필터 버튼의 선택 상태가 약하고, 로그인 role에 따른 상단 메뉴 상태가 즉시 명확하지 않다.
- 영향도: 중간. 사용자가 어떤 필터가 적용되었는지, 관리자 메뉴가 왜 보이는지 혼동할 수 있다.
- 수정 방향: 필터 active 스타일과 `aria-pressed`를 강화하고, 관리자 메뉴는 ADMIN일 때만 노출한다.
- 실제 수정 여부: 수정 완료.

## 2. 현실 세계와의 일치

- 현재 문제: 로그인 페이지에 `test01 / 1234`, `admin / 1234` 같은 시연 계정이 직접 노출되어 실제 서비스 화면처럼 보이지 않는다. footer에도 기술 스택 문구가 노출된다.
- 영향도: 높음. 시연 완성도가 낮아 보인다.
- 수정 방향: 로그인 화면에는 일반적인 로그인 안내만 남기고, footer는 BeautyLens 중심 문구로 정리한다.
- 실제 수정 여부: 수정 완료.

## 3. 사용자 제어와 자유

- 현재 문제: 로그아웃 버튼은 있으나 관리자 메뉴가 USER에게도 보이면 잘못된 경로로 들어갈 가능성이 있다.
- 영향도: 중간. 사용자가 접근 불가 화면으로 이동하는 흐름이 많아진다.
- 수정 방향: USER와 미로그인 상태에서는 관리자 메뉴를 숨기고, 직접 URL 접근 시 기존 안내를 유지한다.
- 실제 수정 여부: 수정 완료.

## 4. 일관성과 표준

- 현재 문제: 필터 버튼의 active 스타일이 일반 버튼과 크게 구분되지 않는다. 현재 페이지 nav active도 서버 설정과 URL 상태가 어긋날 여지가 있다.
- 영향도: 중간.
- 수정 방향: 필터/네비게이션 active 스타일을 공통 CSS로 강화하고, JS에서 현재 URL 기준 active 상태를 보정한다.
- 실제 수정 여부: 수정 완료.

## 5. 오류 예방

- 현재 문제: 상품 카드 높이가 들쭉날쭉해 버튼 위치가 바뀌면 사용자가 잘못 클릭하거나 시선 이동이 커진다.
- 영향도: 중간.
- 수정 방향: 상품명과 추천 이유를 line-clamp로 제한하고, 버튼 영역을 카드 하단에 고정한다.
- 실제 수정 여부: 수정 완료.

## 6. 기억보다 인식

- 현재 문제: 관리자 접근 가능 여부와 필터 선택 상태를 사용자가 기억해야 한다.
- 영향도: 중간.
- 수정 방향: ADMIN에게만 관리자 메뉴를 표시하고, 선택된 필터는 명확한 시각 상태와 `aria-pressed`로 인식 가능하게 한다.
- 실제 수정 여부: 수정 완료.

## 7. 유연성과 효율성

- 현재 문제: 추천 카드의 긴 텍스트가 목록 스캔을 방해한다.
- 영향도: 중간.
- 수정 방향: 카드 내 정보 계층을 유지하되 제목 2줄, 추천 이유 3줄로 제한해 스캔성을 높인다.
- 실제 수정 여부: 수정 완료.

## 8. 미니멀하고 심미적인 디자인

- 현재 문제: footer의 기술 스택 문구와 로그인 페이지의 시연 계정 문구가 화면 목적에 비해 과하다.
- 영향도: 높음.
- 수정 방향: 기술 문구를 제거하고 사용자 친화적인 짧은 문구로 정리한다.
- 실제 수정 여부: 수정 완료.

## 9. 오류 인식/진단/복구

- 현재 문제: 댓글 삭제 실패 시 단순 alert만 표시되고 권한 문제 설명이 부족하다.
- 영향도: 낮음. 서버 권한 정책은 정상이나 실패 안내가 둔하다.
- 수정 방향: 삭제 실패 문구를 권한/재시도 가능성을 포함한 안내로 정리한다.
- 실제 수정 여부: 부분 수정 완료.

## 10. 도움말과 문서화

- 현재 문제: 화면에는 최소 안내만 필요하지만, 시연 계정과 대체 동선은 문서에 있어야 한다.
- 영향도: 낮음.
- 수정 방향: 화면에서는 계정 노출을 제거하고, 시연 계정은 `docs/17_demo_script.md`에 유지한다.
- 실제 수정 여부: 수정 완료.

## 최종 보완 항목

- 관리자 메뉴 ADMIN 전용 노출
- 로그인 페이지 시연 계정 제거
- footer 기술 문구 제거
- 상품 카드 제목/추천 이유 말줄임 및 버튼 하단 정렬
- 추천 필터 active 스타일 및 접근성 상태 강화
- URL 기준 nav active 보정
- 직접 관리자 URL 접근 시 권한 안내 유지

## 2026-06-30 추가 보완: 실제 서비스형 UX 정리

- 메인 화면: 큰 `로그인`, `관리자 페이지` CTA를 제거하고 `추천 받기`, `피부 타입 선택하기` 중심으로 변경했다. 관리자/soft delete 같은 내부 구현 문구는 메인에서 제거했다.
- 관리자 메뉴: 미로그인과 일반 USER에게는 숨기고, ADMIN 로그인 상태에서만 표시하도록 공통 네비게이션을 보완했다.
- 추천 페이지: 건성/지성/복합성/민감성/중성 버튼을 추가해 마이페이지 정보를 바꾸지 않고 임시 추천 기준을 즉시 변경할 수 있게 했다.
- 추천 API: 기존 `/api/recommendations/me`는 유지하고, `/api/recommendations?skinType={type}&size=20`을 추가해 기존 추천 서비스 로직을 재사용했다.
- 마이페이지: `내가 남긴 의견` 섹션을 추가해 작성 댓글, 상태, 상품 이동, 본인 삭제를 확인할 수 있게 했다.
- 댓글 API: `/api/members/me/comments`를 추가해 로그인 회원의 댓글 목록을 최신순으로 반환한다.
- 카드/필터: 상품 카드 높이 안정화, 선택 필터 active 표시, 현재 추천 기준 문구를 보강했다.
