# BeautyLens 이해하기

이 문서는 BeautyLens 프로젝트를 기능 중심으로 빠르게 이해하기 위한 정리 문서입니다. 코드를 먼저 보는 대신, 사용자가 화면에서 무엇을 하고 그 행동이 어떤 데이터와 기능으로 이어지는지를 기준으로 설명합니다.

## 1. BeautyLens는 무엇인가

BeautyLens는 크롤링한 화장품 상품과 리뷰 데이터를 기반으로, 사용자의 피부 타입에 맞는 상품을 추천하고 상품 평가와 댓글을 통해 사용자 의견을 확인할 수 있는 화장품 추천 웹 서비스입니다.

단순히 상품 목록만 보여주는 쇼핑몰이 아니라 다음 세 가지 데이터를 함께 사용합니다.

- 외부 데이터: 크롤링으로 수집한 상품 정보와 외부 리뷰
- 사용자 데이터: 찜, 별점, 자극 여부, 재구매 의사, 댓글, 최근 본 상품
- 관리자 데이터: 상품 숨김, 추천 제외, 품질 상태, 운영 메모, 신고 처리, 활동 로그

즉, BeautyLens의 핵심은 “외부 리뷰 기반 추천”에 “사이트 내부 사용자 반응”과 “관리자 운영 기능”을 붙여 실제 서비스처럼 보이게 만든 구조입니다.

## 2. 전체 흐름 한눈에 보기

전체 서비스 흐름은 다음과 같습니다.

1. 크롤링한 상품과 리뷰 데이터를 DB에 넣는다.
2. 사용자는 회원가입 또는 로그인을 한다.
3. 사용자는 피부 타입과 피부 관심사를 설정한다.
4. 추천 페이지에서 피부 타입 기준 추천 상품을 본다.
5. 상품 상세에서 외부 리뷰 분석, 사이트 사용자 평가, 회원 의견을 확인한다.
6. 사용자는 찜, 별점, 자극 여부, 재구매 의사, 댓글, 신고 같은 행동을 남긴다.
7. 마이페이지에서 자신의 활동을 확인하고 관리한다.
8. 관리자는 상품 노출, 추천 제외, 댓글/신고, 운영 로그를 관리한다.
9. 추천 화면은 기존 리뷰 점수와 내부 사용자 반응, 관리자 운영 상태를 함께 고려한다.

## 3. 화면 구조

사용자 화면은 다음 흐름으로 보면 됩니다.

- `/`  
  서비스 소개, 추천 받기, 상품 둘러보기 진입

- `/login`  
  아이디/비밀번호 로그인, 얼굴 로그인

- `/signup`  
  회원가입, 피부 타입, 피부 관심사 설정

- `/products`  
  전체 상품 탐색, 검색/필터, 찜, 상세 이동

- `/recommend`  
  피부 타입 기준 추천 상품 확인, 추천 숨기기, 상세 이동

- `/products/{productId}`  
  상품 요약, 추천 근거, 외부 리뷰 분석, 사이트 사용자 평가, 내 평가, 회원 의견

- `/mypage`  
  프로필, 피부 타입/관심사, 얼굴 로그인, 찜한 상품, 평가한 상품, 최근 본 상품, 내가 남긴 의견, 추천 피드백

관리자 화면은 다음 흐름으로 보면 됩니다.

- `/admin` 또는 `/admin/dashboard`  
  운영 현황 요약

- `/admin/products`  
  상품 숨김/복구, 추천 제외/포함, 품질 상태, 운영 메모

- `/admin/comments`  
  회원 댓글 관리

- `/admin/comment-reports`  
  신고 댓글 확인과 처리

- `/admin/logs`  
  관리자 작업 로그 확인

## 4. DB를 기능별로 이해하기

BeautyLens의 DB는 “한 테이블에 전부 넣는 구조”가 아니라 기능 성격에 따라 나눠져 있습니다.

### 회원

주요 테이블:

- `BL_MEMBERS`

역할:

- 로그인 아이디, 비밀번호, 닉네임, 피부 타입, 피부 관심사, 권한을 저장합니다.
- 피부 타입은 추천의 큰 기준입니다.
- 피부 관심사는 추천 이유를 더 개인화하기 위한 보조 기준입니다.

예:

- 피부 타입: 건성, 지성, 복합성, 민감성, 중성
- 피부 관심사: 보습, 진정, 트러블, 모공, 피지, 각질 등

### 상품

주요 테이블:

- `BL_PRODUCTS`

역할:

- 크롤링한 상품의 기본 정보를 저장합니다.
- 상품명, 브랜드, 플랫폼, 이미지, 상품 URL, 추천 점수, 리뷰 기반 태그 등이 들어갑니다.

이 테이블은 원본 상품 데이터에 가깝기 때문에 관리자 숨김이나 추천 제외를 하더라도 직접 삭제하지 않습니다.

### 크롤링 리뷰

주요 테이블:

- `BL_REVIEWS`

역할:

- 외부 플랫폼에서 수집한 리뷰를 저장합니다.
- 상품과 `PRODUCT_ID`로 연결됩니다.
- 상품 상세의 “외부 리뷰 분석” 영역에서 사용됩니다.

회원 댓글과 분리한 이유:

- 크롤링 리뷰는 외부 데이터입니다.
- 회원 댓글은 BeautyLens 내부 사용자가 남긴 의견입니다.
- 두 데이터는 출처와 성격이 다르기 때문에 분리해서 관리합니다.

### 회원 댓글

주요 테이블:

- `BL_PRODUCT_COMMENTS`

역할:

- 회원이 상품 상세에서 남기는 자유 의견을 저장합니다.
- 댓글 작성, 수정, 삭제 기능에 사용됩니다.
- 삭제는 실제 삭제가 아니라 상태값으로 관리하는 방식입니다.

상품 평가와 분리한 이유:

- 댓글은 자유 의견입니다.
- 별점/자극 여부/재구매 의사는 정량 평가입니다.
- 두 기능을 섞으면 사용자가 같은 내용을 두 번 쓰는 것처럼 느끼기 때문에 화면과 테이블을 분리했습니다.

### 찜

주요 테이블:

- `BL_PRODUCT_FAVORITES`

역할:

- 사용자가 관심 있는 상품을 저장합니다.
- 상품 카드와 상품 상세에서 찜하기/찜 해제를 할 수 있습니다.
- 마이페이지의 “찜한 상품”에서 다시 볼 수 있습니다.

### 상품 평가

주요 테이블:

- `BL_PRODUCT_RATINGS`

역할:

- 사용자의 별점, 자극 여부, 재구매 의사, 짧은 평가 메모를 저장합니다.
- 사이트 내부 사용자 평가 요약에 사용됩니다.
- 상품 상세의 “내 평가 남기기”와 마이페이지의 “내가 평가한 상품”에서 사용됩니다.

평가 항목:

- 별점
- 자극 여부
- 재구매 의사
- 짧은 평가 메모

### 추천 피드백

주요 테이블:

- `BL_RECOMMENDATION_FEEDBACK`

역할:

- 추천 결과가 마음에 드는지, 별로인지, 다시 보고 싶지 않은지 저장합니다.
- 추천 페이지에서 “추천 숨기기” 같은 보조 행동으로 사용됩니다.
- `NOT_INTERESTED`로 저장된 상품은 추천 화면에서 다시 보이지 않도록 처리합니다.

### 최근 본 상품과 사용자 행동

주요 테이블:

- `BL_USER_PRODUCT_EVENTS`

역할:

- 사용자가 상품 상세를 본 기록, 찜, 평가, 추천 피드백 같은 행동 로그를 저장합니다.
- 마이페이지의 최근 본 상품에 사용됩니다.
- 나중에 추천 품질을 개선할 수 있는 사용자 행동 데이터입니다.

### 댓글 신고

주요 테이블:

- `BL_COMMENT_REPORTS`

역할:

- 사용자가 부적절한 댓글을 신고하면 신고 내역을 저장합니다.
- 관리자는 신고 목록을 보고 처리할 수 있습니다.

### 얼굴 로그인

주요 테이블:

- `BL_FACE_CREDENTIALS`

역할:

- 회원의 얼굴 로그인 등록 정보를 저장합니다.
- 일반 회원 정보와 분리해서 관리합니다.
- 로그인 화면에서는 얼굴 로그인을 비밀번호 로그인의 보조 수단으로 제공합니다.

### 관리자 상품 운영 상태

주요 테이블:

- `BL_PRODUCT_ADMIN_FLAGS`

역할:

- 상품 숨김/복구
- 추천 제외/포함
- 품질 상태 관리
- 운영 메모 저장

중요한 점:

- 원본 상품인 `BL_PRODUCTS`를 직접 삭제하지 않습니다.
- 숨김이나 추천 제외는 운영 상태 테이블에서 관리합니다.
- 이렇게 해야 나중에 복구할 수 있고, 원본 크롤링 데이터도 보존됩니다.

### 관리자 활동 로그

주요 테이블:

- `BL_ADMIN_AUDIT_LOGS`

역할:

- 관리자가 상품을 숨김 처리하거나 추천 제외하거나 신고를 처리한 기록을 저장합니다.
- 누가, 언제, 어떤 대상을, 어떻게 바꿨는지 추적할 수 있습니다.

## 5. 피부 타입과 피부 관심사

이 부분이 헷갈릴 수 있습니다.

### 피부 타입

피부 타입은 추천의 큰 기준입니다.

예:

- 건성
- 지성
- 복합성
- 민감성
- 중성

추천 페이지에서는 이 값을 기준으로 상품을 우선 추천합니다.

### 피부 관심사

피부 관심사는 사용자가 특히 신경 쓰는 세부 고민입니다.

예:

- 보습
- 진정
- 트러블
- 여드름
- 건조
- 민감
- 홍조
- 모공
- 피지
- 각질
- 미백
- 장벽

사용자는 2~4개 정도만 선택합니다.

이 값은 상품의 태그와 비교됩니다. 상품 태그와 사용자의 관심사가 맞으면 추천 이유에 반영됩니다.

예:

사용자 관심사:

- 보습
- 진정
- 장벽

상품 태그:

- 보습
- 진정

추천 이유:

- 피부 고민 키워드와 관련된 태그가 포함되어 있습니다.

즉, 피부 관심사는 상품 점수를 완전히 바꾸는 핵심 조건이라기보다는 “왜 이 상품이 나에게 맞는지 설명하는 보조 근거”로 이해하면 됩니다.

## 6. 추천 기능 흐름

추천 기능은 다음 순서로 동작합니다.

1. 사용자가 추천 페이지에 들어간다.
2. 로그인 상태라면 회원의 피부 타입과 피부 관심사를 읽는다.
3. 피부 타입 기준으로 추천 상품을 조회한다.
4. 관리자가 숨긴 상품은 제외한다.
5. 관리자가 추천 제외한 상품은 추천 결과에서 제외한다.
6. 사용자가 “관심 없음” 처리한 상품도 제외한다.
7. 상품 카드에는 핵심 정보와 추천 이유를 보여준다.
8. 깊은 평가는 상품 상세에서 하도록 유도한다.

추천에서 사용하는 데이터:

- `BL_PRODUCTS.RECOMMENDATION_SCORE`
- 상품의 피부 타입 기준 점수
- 외부 리뷰 기반 긍정/주의 신호
- 사이트 내부 별점
- 찜 수
- 조회 수
- 댓글 수
- 추천 피드백
- 관리자 숨김/추천 제외 상태

중요한 점:

- 기존 리뷰 기반 추천 점수는 유지합니다.
- 사이트 내부 사용자 반응은 보조 지표로 사용합니다.
- 숨김 상품은 사용자 화면과 사용자 API에서 unavailable로 처리합니다.
- 추천 제외 상품은 일반 상품 목록에는 나올 수 있지만 추천 화면에서는 제외됩니다.

## 7. 상품 목록 기능

화면:

- `/products`

사용자가 할 수 있는 일:

- 상품 검색
- 플랫폼 필터
- 이미지 있는 상품만 보기
- 정렬
- 찜하기
- 상품 상세로 이동

상품 카드에서 중요한 행동은 두 개입니다.

- 찜하기
- 자세히 보기

별점, 자극 여부, 재구매 의사 같은 깊은 평가는 카드에서 바로 받지 않고 상품 상세에서 받습니다. 사용자가 상품 정보를 충분히 본 뒤 평가하게 하기 위한 구조입니다.

## 8. 상품 상세 기능

화면:

- `/products/{productId}`

상품 상세는 다음 영역으로 나뉩니다.

### 상품 요약

보여주는 정보:

- 상품 이미지
- 브랜드 또는 플랫폼
- 상품명
- 추천 점수
- 서비스 점수
- 외부 상품 페이지 이동
- 찜하기

### 추천 근거

보여주는 정보:

- 피부 타입 기준 추천 이유
- 긍정 신호
- 주의 신호
- 내부 반응 부족 안내
- 추천 피드백

### 외부 리뷰 분석

보여주는 정보:

- 크롤링 리뷰 기반 긍정 리뷰
- 크롤링 리뷰 기반 주의 리뷰
- 리뷰 감성 요약

여기에는 회원 댓글을 섞지 않습니다.

### 사이트 사용자 평가

보여주는 정보:

- 내부 평균 별점
- 평가 수
- 자극 여부 요약
- 재구매 의사 요약

평가가 부족하면 “아직 사이트 사용자 평가가 부족합니다.” 같은 문구를 보여줍니다.

### 내 평가 남기기

사용자가 남길 수 있는 것:

- 별점
- 자극 여부
- 재구매 의사
- 짧은 평가 메모

이 영역은 정량 평가용입니다.

### 회원 의견

사용자가 할 수 있는 것:

- 댓글 작성
- 댓글 수정
- 댓글 삭제
- 댓글 신고

이 영역은 자유 의견용입니다. 평가와 댓글을 분리해서, 같은 내용을 두 번 쓰는 느낌이 들지 않도록 구성했습니다.

## 9. 마이페이지 기능

화면:

- `/mypage`

마이페이지는 사이드바/탭 구조입니다.

주요 메뉴:

- 내 프로필
- 피부 타입
- 얼굴 로그인
- 찜한 상품
- 내가 평가한 상품
- 최근 본 상품
- 내가 남긴 의견
- 추천 피드백
- 계정 설정

마이페이지에서 중요한 점:

- 사용자는 자신의 활동 데이터를 한곳에서 확인할 수 있습니다.
- 찜한 상품을 다시 볼 수 있습니다.
- 평가한 상품을 수정하거나 삭제할 수 있습니다.
- 최근 본 상품을 확인할 수 있습니다.
- 추천에서 숨긴 상품을 관리할 수 있습니다.
- 얼굴 로그인 등록 상태를 확인할 수 있습니다.

## 10. 댓글 기능

댓글은 상품 상세의 “회원 의견” 영역에서 사용합니다.

사용자가 할 수 있는 일:

- 댓글 작성
- 내 댓글 수정
- 내 댓글 삭제
- 다른 댓글 신고

관리자가 할 수 있는 일:

- 댓글 목록 조회
- 댓글 삭제
- 신고 댓글 확인
- 신고 처리
- 댓글 복구

댓글은 상품 평가와 다릅니다.

- 상품 평가는 별점/자극/재구매 의사처럼 정량 데이터입니다.
- 댓글은 다른 회원에게 보이는 자유 의견입니다.

## 11. 평가 기능

평가는 상품 상세의 “내 평가 남기기” 영역에서 사용합니다.

사용자가 남길 수 있는 것:

- 별점
- 자극 여부
- 재구매 의사
- 짧은 평가 메모

평가는 다음에 사용됩니다.

- 상품 상세의 사이트 사용자 평가 요약
- 마이페이지 내가 평가한 상품
- 추천 품질 보조 지표

평가 수정/삭제도 지원합니다.

## 12. 찜 기능

찜은 가장 가벼운 관심 표현입니다.

사용 가능한 화면:

- 상품 목록
- 추천 페이지
- 상품 상세
- 마이페이지

찜 데이터는 `BL_PRODUCT_FAVORITES`에 저장됩니다.

찜은 추천 품질을 판단할 때 보조 신호로 사용할 수 있습니다. 예를 들어 어떤 상품이 많이 찜되면 사이트 내부 반응이 좋은 상품으로 볼 수 있습니다.

## 13. 최근 본 상품

사용자가 상품 상세에 들어가면 최근 본 상품 이벤트가 기록됩니다.

관련 테이블:

- `BL_USER_PRODUCT_EVENTS`

마이페이지에서는 이 데이터를 기준으로 최근 본 상품을 보여줍니다.

숨김 상품은 최근 본 상품 목록에서도 제외됩니다. 관리자가 숨긴 상품이 사용자 활동 목록에 계속 남아 있으면 실제 서비스 흐름상 어색하기 때문입니다.

## 14. 얼굴 등록과 얼굴 로그인

얼굴 로그인은 비밀번호 로그인을 대체하는 주 기능이라기보다 보조 로그인 수단입니다.

### 얼굴 등록

사용자는 마이페이지에서 얼굴 정보를 등록합니다.

등록 흐름:

1. 안내
2. 정면 촬영
3. 왼쪽 촬영
4. 오른쪽 촬영
5. 위쪽 촬영
6. 아래쪽 촬영
7. 등록 완료

사용자 화면에는 기술 모델명이나 서버 내부 용어가 나오지 않게 했습니다.

### 얼굴 로그인

로그인 화면에서 얼굴로 로그인할 수 있습니다.

흐름:

1. 얼굴로 로그인 클릭
2. 카메라 모달 열림
3. 얼굴을 화면 중앙에 맞춤
4. 자동으로 얼굴 확인 시도
5. 로그인 정보 확인
6. 성공 시 로그인
7. 실패 시 비밀번호 로그인 안내

비슷한 얼굴 때문에 계정 식별이 애매한 경우에는 바로 로그인시키지 않고 추가 각도 촬영을 요구합니다. 추가 확인 후에도 확실하지 않으면 보수적으로 로그인 실패 처리하고 비밀번호 로그인을 안내합니다.

## 15. 관리자 기능

관리자 기능은 단순 관리 목록이 아니라 운영 콘솔 흐름으로 구성되어 있습니다.

### 관리자 대시보드

화면:

- `/admin`
- `/admin/dashboard`

보여주는 것:

- 전체 상품 수
- 노출 상품 수
- 숨김 상품 수
- 이미지 없는 상품 수
- 댓글 수
- 신고 대기 수
- 내부 평가 수
- 평균 내부 별점
- 얼굴 등록 회원 수
- 최근 운영 로그

### 상품 관리

화면:

- `/admin/products`

관리자가 할 수 있는 일:

- 상품 검색
- 플랫폼 필터
- 이미지 없음 필터
- 상품 숨김/복구
- 추천 제외/포함
- 품질 상태 저장
- 운영 메모 저장
- 상품 상세 바로가기

중요한 정책:

- 상품을 실제 삭제하지 않습니다.
- 숨김은 사용자 화면에서 안 보이게 하는 운영 상태입니다.
- 추천 제외는 추천 영역에서만 제외하는 운영 상태입니다.

### 댓글 관리

화면:

- `/admin/comments`

관리자가 할 수 있는 일:

- 댓글 목록 확인
- 댓글 삭제
- 삭제된 댓글 상태 확인
- 필요 시 복구

### 신고 관리

화면:

- `/admin/comment-reports`

관리자가 할 수 있는 일:

- 신고된 댓글 목록 확인
- 신고 사유 확인
- 신고 처리
- 댓글 복구

### 운영 로그

화면:

- `/admin/logs`

보여주는 것:

- 관리자 ID
- 작업 종류
- 대상 종류
- 대상 ID
- 변경 전/후 값
- 작업 시간

이 기능은 “관리자가 무엇을 바꿨는지” 추적하기 위한 기능입니다.

## 16. 숨김 상품과 추천 제외 상품의 차이

이 둘은 다릅니다.

### 상품 숨김

숨김 상품은 사용자에게 보이면 안 됩니다.

적용되는 곳:

- 상품 목록
- 상품 상세
- 추천 페이지
- 마이페이지 찜/평가/최근 본 상품
- 사용자 interaction API
- 댓글 작성 API

즉, 숨김 상품은 사용자 입장에서는 unavailable 상태입니다.

### 추천 제외

추천 제외 상품은 추천 결과에서만 제외됩니다.

적용되는 곳:

- 추천 페이지
- 추천 fallback 결과

일반 상품 목록에서는 정책상 계속 보일 수 있습니다. 상품 자체가 나쁜 것은 아니지만 추천 영역에만 노출하지 않는 상황이 있을 수 있기 때문입니다.

## 17. 서비스 점수와 추천 점수

BeautyLens에는 두 종류의 점수가 있습니다.

### 추천 점수

외부 리뷰와 상품 데이터를 기반으로 계산된 기존 추천 점수입니다.

### 서비스 점수

BeautyLens 내부 사용자 반응을 보조로 반영한 점수입니다.

반영 가능한 데이터:

- 내부 별점
- 같은 피부 타입 사용자 평가
- 찜 수
- 조회 수
- 댓글 수
- 추천 좋아요/싫어요
- 관심 없음
- 관리자 추천 제외
- 이미지 없음 같은 품질 신호

중요한 점:

- 기존 추천 점수를 없애지 않았습니다.
- 서비스 점수는 보조 지표로 따로 보여줍니다.
- 내부 평가 데이터가 부족하면 부족하다고 안내합니다.

## 18. 왜 테이블을 이렇게 나눴나

처음에는 상품과 리뷰만 있으면 추천 사이트처럼 보일 수 있습니다. 하지만 실제 서비스처럼 보이려면 사용자 행동과 관리자 운영 데이터가 필요합니다.

그래서 데이터를 다음처럼 나눴습니다.

- 상품 원본: `BL_PRODUCTS`
- 외부 리뷰: `BL_REVIEWS`
- 회원 댓글: `BL_PRODUCT_COMMENTS`
- 찜: `BL_PRODUCT_FAVORITES`
- 평가: `BL_PRODUCT_RATINGS`
- 추천 피드백: `BL_RECOMMENDATION_FEEDBACK`
- 최근 본 상품/행동 로그: `BL_USER_PRODUCT_EVENTS`
- 관리자 상품 운영 상태: `BL_PRODUCT_ADMIN_FLAGS`
- 신고: `BL_COMMENT_REPORTS`
- 관리자 로그: `BL_ADMIN_AUDIT_LOGS`
- 얼굴 로그인 정보: `BL_FACE_CREDENTIALS`

이렇게 나눈 이유는 기능이 늘어날수록 한 테이블에 모든 것을 넣으면 관리가 어려워지기 때문입니다.

특히 상품 숨김/추천 제외를 `BL_PRODUCTS`에서 직접 수정하지 않고 별도 운영 상태 테이블로 뺀 이유는 원본 크롤링 데이터를 보존하기 위해서입니다.

## 19. 시연할 때 이해해야 할 핵심 포인트

시연할 때는 다음 순서로 설명하면 이해하기 쉽습니다.

1. BeautyLens는 크롤링 상품/리뷰 기반 화장품 추천 서비스입니다.
2. 사용자는 피부 타입과 피부 관심사를 설정합니다.
3. 추천 페이지에서 피부 타입 기준 상품을 봅니다.
4. 상품 상세에서 외부 리뷰 분석과 내부 사용자 평가를 분리해서 봅니다.
5. 사용자는 찜, 평가, 댓글, 신고 같은 행동을 남깁니다.
6. 마이페이지에서 내 활동을 관리합니다.
7. 얼굴 등록/얼굴 로그인으로 로그인 편의 기능을 제공합니다.
8. 관리자는 상품 숨김, 추천 제외, 댓글/신고, 운영 로그를 관리합니다.
9. 원본 상품/리뷰는 삭제하지 않고 운영 상태를 별도 테이블로 관리합니다.
10. 추천은 외부 리뷰 점수와 내부 사용자 반응을 분리해서 보여줍니다.

## 20. 헷갈리기 쉬운 부분 정리

### 크롤링 리뷰와 회원 댓글은 다르다

크롤링 리뷰는 외부 플랫폼에서 가져온 데이터입니다. 회원 댓글은 BeautyLens 사용자가 직접 남긴 의견입니다.

### 피부 타입과 피부 관심사는 다르다

피부 타입은 추천의 큰 기준입니다. 피부 관심사는 추천 이유를 더 개인화하는 보조 기준입니다.

### 찜과 추천 피드백은 다르다

찜은 관심 있는 상품을 저장하는 기능입니다. 추천 피드백은 추천 결과가 맞는지 아닌지를 알려주는 기능입니다.

### 별점과 댓글은 다르다

별점은 내부 평가 요약에 쓰이는 정량 데이터입니다. 댓글은 다른 회원에게 보이는 자유 의견입니다.

### 숨김과 추천 제외는 다르다

숨김은 사용자 화면 전체에서 제외합니다. 추천 제외는 추천 화면에서만 제외합니다.

### 얼굴 로그인은 보조 수단이다

기본 로그인은 아이디/비밀번호입니다. 얼굴 로그인은 편의 기능이며, 확실하지 않은 경우 비밀번호 로그인을 안내합니다.

발표 전에 필요한 실행 순서는 다음과 같습니다.

1. Oracle DB 실행

## 22. 이 프로젝트를 한 문장으로 설명하면

BeautyLens는 외부 화장품 리뷰 데이터를 기반으로 피부 타입별 추천을 제공하고, 사이트 내부 사용자 평가와 관리자 운영 기능을 결합해 실제 서비스 흐름처럼 구성한 Spring MVC 기반 화장품 추천 웹 서비스입니다.
