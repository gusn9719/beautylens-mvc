## 1. 시연 전 준비

1. Tomcat 실행 상태 확인
2. 브라우저에서 아래 URL 접속 확인
3. 일반 회원과 관리자 계정 로그인 가능 여부 확인
4. `/api/health`에서 `db=ok` 확인
5. 시연 중 사용할 상품 상세 페이지는 추천 카드에서 직접 클릭한다.

확인 URL:

```text
http://localhost:8088/beautylens-mvc/api/health
```

기대 결과:

```text
HTTP 200
db=ok
```

## 5. 5분 시연 순서

1. 메인 페이지 접속
   - `http://localhost:8088/beautylens-mvc/`
   - BeautyLens가 피부 타입과 리뷰 데이터를 기반으로 화장품을 추천하는 서비스라고 소개

2. 추천 페이지 이동
   - `/recommend`
   - 건성, 지성, 복합성, 민감성, 중성 버튼으로 추천 기준을 즉시 바꿀 수 있음을 설명

3. 일반 회원 로그인
   - `/login`
   - `test01 / 1234`

4. 추천 페이지 이동
   - `/recommend`
   - 추천 상품 카드, 이미지, 브랜드, 추천 점수, 추천 이유 설명

5. 상품 상세 이동
   - 추천 상품 카드의 “자세히 보기” 클릭
   - 상품 이미지, 추천 점수, 리뷰 수, 주의 신호 설명

6. 리뷰와 댓글 설명
   - 긍정 리뷰와 주의해서 볼 리뷰가 크롤링 리뷰 기반으로 분리되어 있다고 설명
   - 회원 댓글은 별도 테이블로 관리된다고 설명

7. 댓글 작성과 본인 삭제
   - 짧은 댓글 작성
   - 작성한 댓글 삭제
   - 삭제 후 일반 댓글 목록에서 사라지는 것을 확인

8. 마이페이지 내 의견 확인
   - `/mypage`
   - 내가 남긴 의견 목록과 상품 보러가기 동선을 설명

9. 관리자 로그인
   - 로그아웃
   - `admin / 1234` 로그인

10. 관리자 대시보드와 댓글 관리
   - `/admin` 통계 카드 설명
   - `/admin/comments`에서 댓글 목록과 삭제 기능 설명

11. 마무리
   - 추천, 리뷰 분석, 회원 의견, 관리자 관리까지 하나의 Spring MVC 서비스로 연결했다고 정리

## 7. 사용자가 설명할 멘트

### 메인 페이지

```text
BeautyLens는 피부 타입과 리뷰 데이터를 기반으로 화장품을 추천하는 서비스입니다.
크롤링된 상품/리뷰 데이터와 회원 정보를 연결해서 추천 상품, 추천 이유, 리뷰 신호를 한 화면에서 보여줍니다.
```

### 로그인

```text
시연용 일반 회원 test01 계정으로 로그인하겠습니다.
이 계정에는 피부 타입 정보가 있어 개인화 추천 API를 바로 호출할 수 있습니다.
```

### 추천 페이지

```text
추천 화면에서는 상품 이미지, 브랜드, 플랫폼, 추천 점수, 추천 이유를 카드 형태로 보여줍니다.
상단에서 피부 타입을 바로 선택하면 회원 정보를 수정하지 않고도 해당 피부 기준의 추천 결과를 확인할 수 있습니다.
이미지가 있는 상품만 보거나 플랫폼별로 필터링할 수 있습니다.
```

### 마이페이지

```text
마이페이지에서는 내 피부 타입과 관심사를 관리할 수 있고, 내가 상품에 남긴 의견도 함께 확인할 수 있습니다.
각 의견에서 상품 상세로 이동할 수 있으며, 게시 중인 내 의견은 이 화면에서도 삭제할 수 있습니다.
```

### 상품 상세

```text
상세 화면에서는 상품 기본 정보와 함께 긍정 리뷰, 주의해서 볼 리뷰, 회원 댓글을 분리해서 보여줍니다.
크롤링 리뷰와 회원 댓글은 서로 다른 데이터로 관리됩니다.
```

### 댓글

```text
로그인한 회원은 상품에 댓글을 작성할 수 있고, 본인이 작성한 댓글은 직접 삭제할 수 있습니다.
삭제는 실제 삭제가 아니라 soft delete 방식이라 관리자 화면에서 이력을 확인할 수 있습니다.
```

### 관리자

```text
관리자는 전체 상품, 리뷰, 회원, 댓글 현황을 대시보드에서 확인할 수 있습니다.
댓글 관리 화면에서는 ACTIVE와 DELETED 상태를 필터링하고, 부적절한 댓글을 soft delete 처리할 수 있습니다.
```

### 마무리

```text
이 프로젝트는 Oracle, MyBatis XML, Spring MVC, JSP를 사용해 추천 API와 관리자 기능까지 연결한 시연 가능한 웹 서비스입니다.
React 없이 JSP와 vanilla JavaScript만으로 오늘 시연 가능한 형태로 완성했습니다.
```

## 8. 문제가 생겼을 때 대체 동선

### 얼굴 로그인 서버가 꺼져 있을 때

1. 별도 터미널에서 Python 서버 실행
   ```powershell
   cd D:\Lecture\spring-workspace\beautylens-mvc\face_auth_server
   python -m uvicorn app:app --host 127.0.0.1 --port 8090
   ```
2. 브라우저에서 확인
   ```text
   http://127.0.0.1:8090/health
   ```
3. 얼굴 로그인이 계속 실패하면 비밀번호 로그인을 사용

설명 멘트:

```text
얼굴 인증은 Spring 서버와 별도로 로컬 Python FastAPI 서버가 임베딩 추출을 담당합니다.
Python 서버가 꺼져 있으면 얼굴 등록과 얼굴 로그인만 실패하고, 기존 비밀번호 로그인은 그대로 사용할 수 있습니다.
```

### 웹캠 권한이 거부될 때

1. 브라우저 주소창의 카메라 권한 확인
2. localhost 접속인지 확인
3. 다른 앱이 카메라를 사용 중인지 확인
4. 비밀번호 로그인으로 대체

설명 멘트:

```text
얼굴 등록과 로그인은 브라우저의 웹캠 권한이 필요합니다.
권한이 거부되어도 기존 아이디/비밀번호 로그인은 정상 동작합니다.
```

### 추천 페이지가 비어 보일 때

1. `/login`에서 `test01 / 1234`로 다시 로그인
2. `/mypage`에서 skinType이 설정되어 있는지 확인
3. `/api/recommendations/me?size=20` 직접 확인

설명 멘트:

```text
추천 API는 로그인 회원의 피부 타입을 기준으로 동작합니다.
회원 정보가 없거나 세션이 만료되면 추천 화면 대신 로그인 안내가 표시됩니다.
```

### 상품 이미지가 늦게 뜰 때

1. 새로고침
2. 이미지 placeholder가 표시되는지 확인
3. 이미지 있는 상품만 보기 필터 사용

설명 멘트:

```text
이미지는 외부 쇼핑몰 og:image URL을 저장해 표시합니다.
일부 상품은 아직 이미지 URL이 없기 때문에 placeholder로 처리했습니다.
```

### 댓글 작성이 실패할 때

1. 로그인 상태 확인
2. 댓글 내용이 비어 있지 않은지 확인
3. 다른 상품 상세에서 다시 시도

설명 멘트:

```text
댓글 작성은 로그인 회원만 가능하고, 빈 내용은 서버에서 차단합니다.
```

### 관리자 화면이 차단될 때

1. 로그아웃
2. `admin / 1234`로 다시 로그인
3. `/admin` 접속

설명 멘트:

```text
관리자 API는 ADMIN role만 접근할 수 있습니다.
일반 회원이나 미로그인 사용자는 403 또는 로그인 안내가 표시됩니다.
```

### Tomcat 배포 직후 404가 보일 때

1. 10~20초 대기
2. 새로고침
3. `/api/health` 확인

설명 멘트:

```text
WAR 교체 직후 Tomcat autoDeploy 중에는 잠시 404가 보일 수 있습니다.
배포가 끝나면 정상 접속됩니다.
```

## 9. 이미지가 없는 상품이 보일 때 설명

```text
현재 전체 1,521개 상품 중 추천 상위 상품 중심으로 일부 imageUrl을 수집했습니다.
이미지가 없는 상품은 데이터가 없는 것이 아니라 이미지 URL 수집이 아직 완료되지 않은 상태입니다.
그래서 화면에서는 이미지 준비 중 placeholder를 보여 주도록 처리했습니다.
```

핵심:
- 상품 데이터는 존재함
- 추천 로직은 정상 동작함
- 이미지 URL만 일부 상품에 없음
- 전체 이미지 수집은 별도 단계로 남겨둠

## 10. 추천 결과가 10개만 나올 때 설명

```text
현재 test01 회원의 피부 타입 기준으로 추천 API가 직접 반환하는 개인화 추천 결과는 10개입니다.
화면에서는 시연 편의를 위해 부족한 카드 수를 추천 점수 상위 상품으로 보완 표시합니다.
추천 API 자체나 추천 알고리즘은 변경하지 않았습니다.
```

핵심:
- API는 정상
- 현재 데이터 분포상 개인화 결과가 10개
- UI는 Product API 결과로 보완 표시
- 추천 로직은 그대로 유지

## 14. 기능 확장 시연 동선

권장 순서:

1. `test01 / 1234`로 로그인한다.
2. `/products`에서 상품 카드의 `찜`을 누른다.
3. 상품 상세로 이동해 `사이트 사용자 평가`에서 별점, 자극 여부, 재구매 의사를 저장한다.
4. `/recommend`에서 추천 카드의 `좋아요`, `별로예요`, `관심 없음` 중 하나를 누른다.
5. `/mypage`로 이동해 찜한 상품, 내가 평가한 상품, 최근 본 상품, 추천 피드백 내역을 보여준다.
6. 상품 상세 댓글에서 신고 버튼을 눌러 신고를 접수한다.
7. `admin / 1234`로 로그인한다.
8. `/admin/products`에서 상품 숨김, 추천 제외, 품질 상태, 운영 메모를 보여준다.
9. `/admin/comment-reports`에서 신고 댓글을 처리하고 댓글 복구를 보여준다.
10. `/admin/logs`에서 관리자 작업 로그가 남는 것을 보여준다.

설명 멘트:

```text
기존 크롤링 리뷰 점수는 그대로 보존하고, 사용자가 사이트 안에서 남긴 찜, 별점, 조회, 추천 피드백은 별도 테이블에 쌓습니다.
추천 카드에는 기존 추천 점수와 서비스 점수를 분리해서 보여주고, 관리자는 상품을 삭제하지 않고 숨김/추천 제외/품질 메모로 운영합니다.
신고 처리와 관리자 로그까지 연결되어 실제 서비스 운영 흐름처럼 설명할 수 있습니다.
```

- `/api/health` DB ok
- Python 얼굴 서버 `/health` 정상
