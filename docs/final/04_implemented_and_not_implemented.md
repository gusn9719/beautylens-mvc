# 구현한 것과 보완할 점

## 구현한 것

### 사용자 기능

- 회원가입, 로그인, 로그아웃
- 상품 목록 조회와 검색/정렬/필터
- 피부 타입 기반 추천
- 상품 상세 조회
- 외부 리뷰 분석 확인
- 사이트 사용자 평가 확인
- 찜하기와 찜 해제
- 최근 본 상품 기록
- 별점, 자극 여부, 재구매 의사 평가 등록/수정/삭제
- 댓글 작성/수정/삭제
- 댓글 신고
- 마이페이지 활동 통합
- 얼굴 등록
- 얼굴 로그인

### 관리자 기능

- 관리자 대시보드
- 상품 관리
- 상품 숨김/복구
- 추천 제외/포함
- 품질 상태 관리
- 운영 메모
- 댓글 관리
- 신고 댓글 처리
- 댓글 복구
- 관리자 활동 로그 조회

### DB/데이터 활용

- 크롤링 상품 데이터 `BL_PRODUCTS` 적재
- 크롤링 리뷰 데이터 `BL_REVIEWS` 적재
- import 결과 `BL_IMPORT_LOGS` 기록
- 외부 리뷰와 회원 댓글 분리
- 사이트 사용자 반응 데이터 별도 저장
- 관리자 운영 상태 별도 저장
- 얼굴 로그인 인증 정보 별도 저장
- 숨김 상품과 추천 제외 상품의 사용자 노출 정책 반영

### UI/UX 개선

- 상품 카드의 행동 버튼을 찜하기/자세히 보기 중심으로 정리
- 추천 피드백은 카드의 보조 UI로 이동
- 상품 상세를 상품 요약, 추천 근거, 외부 리뷰 분석, 사이트 사용자 평가, 내 평가, 회원 의견으로 분리
- 마이페이지를 사이드바/탭 구조로 정리
- 관리자 화면을 사이드바 기반 운영 콘솔 구조로 정리
- 얼굴 로그인에서 진행률, 상태 문구, 자동 확인, 취소 처리, 중복 실행 방지 적용

## 시간상 보완으로 남긴 것

아래 항목은 현재 프로젝트 범위에서는 구현하지 않았거나 제한적으로 구현한 부분입니다.

| 보완 항목 | 현재 상태 | 보완 방향 |
|---|---|---|
| 프론트엔드 분리 | JSP, CSS, vanilla JS 기반으로 구현 | 시간이 더 있으면 프론트엔드 구조를 별도 모듈로 정리 가능 |
| 상품 비교함 | 미구현 | 찜한 상품 또는 선택 상품을 비교하는 화면 추가 가능 |
| 세부 검색/필터 | 키워드, 플랫폼, 이미지 여부, 정렬 중심 | 브랜드, 가격대, 카테고리 세부 필터 추가 가능 |
| 마이페이지 계정 설정 | 안내성 화면 수준 | 비밀번호 변경, 회원 탈퇴 같은 계정 관리 기능 보완 가능 |
| 댓글 신고 사유 선택 모달 | 기본 신고 기능 중심 | 사용자가 신고 사유를 선택하는 모달 추가 가능 |
| 추천 피드백 사유 수집 | 좋아요, 별로예요, 추천 숨기기 중심 | 왜 맞지 않았는지 세부 사유 수집 가능 |
| 모바일 세부 검수 | 기본 반응형 적용 | 실제 기기별 화면 검수와 CSS 조정 필요 |
| 브라우저 자동 테스트 | 미구현 | 주요 시나리오를 자동화 테스트로 보완 가능 |
| 얼굴 인증 보안 정책 | 학습용 프로젝트 범위 구현 | 실제 운영 수준의 보안 정책, 동의 절차, 보관 정책 추가 필요 |

## 구현/미구현 정리 기준

- Controller, Mapper, JSP/JS 연결이 있고 빌드 및 기본 검증을 한 기능은 구현으로 분류했습니다.
- 화면 안내만 있고 실제 API나 DB 저장 흐름이 없는 기능은 보완 항목으로 분류했습니다.
- 댓글 수정과 평가 삭제는 실제 API와 화면이 추가되어 있으므로 구현 기능에 포함했습니다.
- 원본 상품/리뷰 데이터를 삭제하거나 직접 수정하는 기능은 의도적으로 구현하지 않았습니다.
