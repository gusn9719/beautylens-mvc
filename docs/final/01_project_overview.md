# BeautyLens 프로젝트 개요

BeautyLens는 크롤링한 화장품 상품과 리뷰 데이터를 기반으로, 사용자의 피부 타입에 맞는 상품을 추천하고 상품 평가와 댓글을 통해 사용자 의견을 확인할 수 있는 화장품 추천 웹 서비스입니다.

## 프로젝트명

BeautyLens

## 한 줄 소개

크롤링한 화장품 상품/리뷰 데이터와 사이트 내부 사용자 반응 데이터를 함께 활용하는 Spring MVC 기반 화장품 추천 웹 서비스입니다.

## 프로젝트 주제

사용자는 상품 목록과 추천 결과를 탐색하고, 상품 상세에서 외부 리뷰 분석과 사이트 사용자 평가를 확인합니다. 로그인한 사용자는 찜, 별점, 자극 여부, 재구매 의사, 댓글, 신고, 최근 본 상품 같은 활동을 남길 수 있습니다. 관리자는 상품 노출, 추천 제외, 품질 상태, 댓글, 신고, 운영 로그를 관리합니다.

## 개발 배경

처음에는 크롤링한 상품과 리뷰를 화면에 보여주는 것이 중심이었습니다. 하지만 단순 목록/상세만 있으면 실제 서비스처럼 보이기 어렵다고 판단했습니다. 그래서 다음 구조를 추가했습니다.

- 외부 크롤링 리뷰와 사이트 내부 회원 의견을 분리
- 사용자의 찜, 평가, 최근 본 상품, 추천 피드백을 별도 테이블에 저장
- 관리자가 원본 상품 데이터를 삭제하지 않고 노출/추천 상태만 운영
- 얼굴 로그인은 회원 정보와 연결하되 인증 정보는 별도 테이블로 분리

## 사용 데이터

| 구분 | 내용 |
|---|---|
| 상품 데이터 | `product_recommendation_scores.parquet` 기반, `BL_PRODUCTS` 적재 |
| 리뷰 데이터 | `service_reviews.parquet` 기반, `BL_REVIEWS` 적재 |
| 플랫폼 | oliveyoung, musinsa |
| 전처리 데이터 | 추천 점수, 피부 타입 기준 집계, 긍정/부정 리뷰 비율, 주의 신호 |
| 사이트 내부 데이터 | 찜, 평가, 댓글, 신고, 최근 본 상품, 추천 피드백 |

전체 import 보고서 기준으로 상품 1,521건, 리뷰 323,574건을 적재했습니다.

## 전체 서비스 흐름

1. 크롤링/전처리 데이터가 상품/리뷰 테이블에 적재됩니다.
2. 사용자는 상품 목록 또는 피부 타입 추천 페이지에서 상품을 탐색합니다.
3. 상품 상세에서 외부 리뷰 분석, 사이트 사용자 평가, 회원 의견을 분리해서 확인합니다.
4. 로그인 사용자는 찜, 평가, 댓글, 신고, 추천 피드백을 남깁니다.
5. 관리자는 상품 숨김/복구, 추천 제외/포함, 댓글/신고 처리, 운영 로그 확인을 합니다.
6. 추천 화면은 기존 추천 점수에 내부 사용자 반응을 보조 점수로 함께 사용합니다.

## 사용자 흐름

1. 홈에서 서비스 설명을 확인합니다.
2. 상품 목록 또는 추천 페이지로 이동합니다.
3. 피부 타입을 선택하거나 내 피부 타입 기반 추천을 확인합니다.
4. 상품 상세에서 추천 근거, 외부 리뷰 분석, 사이트 사용자 평가를 확인합니다.
5. 상품을 찜하거나 평가를 남기고, 다른 회원에게 보이는 댓글을 작성합니다.
6. 마이페이지에서 내 프로필, 피부 타입, 얼굴 로그인, 찜한 상품, 평가, 최근 본 상품, 의견, 추천 피드백을 관리합니다.

## 관리자 흐름

1. 관리자 계정으로 로그인합니다.
2. 관리자 대시보드에서 전체 상품, 리뷰, 회원, 댓글, 신고, 내부 평가, 얼굴 등록 회원 수를 확인합니다.
3. 상품 관리에서 이미지 없음, 숨김 상태, 추천 제외 상태, 품질 상태를 확인합니다.
4. 문제가 있는 상품은 숨김/복구, 추천 제외/포함, 품질 상태, 운영 메모로 관리합니다.
5. 댓글 관리와 신고 관리에서 댓글 삭제, 복구, 신고 처리를 합니다.
6. 운영 로그에서 관리자 작업 이력을 확인합니다.

## 기술 스택

| 구분 | 사용 기술 |
|---|---|
| Backend | Spring MVC 6.x |
| View | JSP, CSS, vanilla JavaScript |
| DB | Oracle XE |
| Persistence | MyBatis XML Mapper |
| Build/Deploy | Maven WAR, Tomcat 11 |
| 얼굴 인증 | Python FastAPI 서버와 Java HTTP client 연동 |
| 데이터 처리 | Python, pandas, pyarrow, oracledb |

## 핵심 포인트

- 크롤링 리뷰와 회원 댓글을 분리해 데이터 성격을 명확히 했습니다.
- 원본 상품/리뷰 데이터를 직접 삭제하지 않고 관리자 운영 상태 테이블로 노출 여부를 관리했습니다.
- 추천 제외는 상품 삭제가 아니라 추천 영역에서만 제외하는 정책으로 구현했습니다.
- 사이트 내부 사용자 반응 데이터를 추천 보조 근거로 사용할 수 있도록 별도 테이블에 쌓았습니다.
- 마이페이지와 관리자 페이지를 단순 기능 목록이 아니라 계정/운영 대시보드 구조로 재배치했습니다.
