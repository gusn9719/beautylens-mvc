# BeautyLens 서비스 개요 보고서

## 1. 서비스 개요

BeautyLens는 크롤링한 화장품 상품과 리뷰 데이터를 기반으로, 사용자의 피부 타입에 맞는 상품을 추천하고 상품 평가와 댓글을 통해 사용자 의견을 확인할 수 있는 화장품 추천 웹 서비스입니다.

## 2. 개발 배경

크롤링한 데이터가 많았고 이 데이터를 버리기 아까워 이거를 베이스로 추천하고 의견을 자유롭게 나누는 사이트를 만들면 좋지 않을까 해서 만들었습니다.

## 3. 주요 기능

### 3.1 사용자 기능

사용자는 회원가입과 로그인을 통해 사이트의 기능을 사용할 수 있습니다.

주요 기능은 다음과 같습니다.

- 회원가입 및 로그인
  - 사용자가 개인 계정을 만들고 로그인할 수 있습니다.
- 피부 타입과 피부 관심사 설정
  - 사용자가 자신의 피부 타입과 관심 고민을 선택해서 추천 기준으로 사용할 수 있습니다.
- 피부 타입 기반 상품 추천
  - 건성, 지성, 민감성 같은 피부 타입에 맞는 상품을 추천받을 수 있습니다.
- 상품 목록 조회 및 검색/필터
  - 전체 상품을 보고, 상품명이나 플랫폼 같은 조건으로 상품을 찾을 수 있습니다.
- 상품 상세 정보 조회
  - 상품 이미지, 브랜드, 추천 점수, 외부 상품 링크, 리뷰 분석 내용을 볼 수 있습니다.
- 외부 리뷰 기반 긍정/주의 리뷰 확인
  - 크롤링한 리뷰 중 긍정적으로 볼 내용과 주의해서 볼 내용을 나눠서 확인할 수 있습니다.
- 상품 찜하기 및 찜 해제
  - 관심 있는 상품을 저장하고, 취소할 수 있습니다.
- 상품 별점 평가
  - 사용자가 상품 만족도를 별점으로 남길 수 있습니다.
- 자극 여부 평가
  - 화장품을 사용했을 때 자극이 있었는지 남길 수 있습니다.
- 재구매 의사 평가
  - 이 상품을 다시 구매하고 싶은지 남길 수 있습니다.
- 회원 댓글 작성, 수정, 삭제
  - 상품에 대한 자유로운 의견을 작성하고, 본인이 쓴 댓글은 수정하거나 삭제할 수 있습니다.
  - 이 댓글은 로그인하지 않아도 보입니다.
- 댓글 신고
  - 문제가 있어 보이는 댓글을 관리자에게 신고할 수 있습니다.
- 최근 본 상품 확인
  - 상세 화면에서 봤던 상품을 마이페이지에서 다시 볼 수 있습니다.
- 마이페이지에서 내 활동 확인
  - 내가 찜한 상품, 평가한 상품, 작성한 댓글, 최근 본 상품을 한곳에서 확인할 수 있습니다.
- 얼굴 등록 및 얼굴 로그인
  - 얼굴 정보를 등록하면 비밀번호 로그인 외에 얼굴 로그인도 사용할 수 있습니다.

### 3.2 추천 기능

추천 기능은 크롤링 리뷰 기반 점수를 중심으로 동작합니다. 사용자의 피부 타입을 기준으로 상품을 추천하고, 사용자가 설정한 피부 관심사가 상품 태그와 맞을 경우 추천 이유에 반영됩니다.
또한 사이트 내부 사용자 반응인 별점, 찜, 조회, 댓글, 추천 피드백 등을 보조 지표로 활용할 수 있도록 구조를 구성했습니다.
추천 화면에서는 상품을 바로 평가하게 하지 않고, 먼저 상품을 비교하고 상세 화면에서 충분히 확인한 뒤 평가하도록 설계했습니다.

### 3.3 상품 상세 기능

상품 상세 화면은 다음과 같이 구성했습니다.

- 상품 요약
  - 상품 이미지, 브랜드, 상품명, 플랫폼, 외부 상품 링크, 찜하기 버튼을 보여줍니다.
- 추천 근거
  - 이 상품이 왜 추천되었는지 피부 타입과 리뷰 신호 기준으로 보여줍니다.
- 외부 리뷰 분석
  - 크롤링한 리뷰를 바탕으로 긍정 리뷰와 주의 리뷰를 나눠서 보여줍니다.
- 사이트 사용자 평가
  - BeautyLens 회원들이 남긴 별점, 자극 여부, 재구매 의사를 요약해서 보여줍니다.
- 내 평가 남기기
  - 사용자가 별점, 자극 여부, 재구매 의사, 짧은 평가 메모를 남길 수 있습니다.
- 회원 의견
  - 회원이 상품에 대해 자유롭게 의견을 남기고 다른 사용자의 의견도 확인할 수 있습니다.

외부 리뷰 분석은 크롤링한 리뷰 데이터를 기반으로 하고, 회원 의견은 BeautyLens 사용자가 직접 작성한 댓글입니다. 두 데이터는 출처와 목적이 다르기 때문에 화면에서도 분리해서 보여주었습니다.

### 3.4 마이페이지 기능

마이페이지에서는 사용자의 활동을 한곳에서 확인할 수 있도록 구성했습니다.

- 내 프로필
  - 로그인 아이디와 닉네임 같은 기본 회원 정보를 확인할 수 있습니다.
- 피부 타입 및 피부 관심사
  - 추천에 사용할 피부 타입과 관심 고민을 선택할 수 있습니다.
- 얼굴 로그인 설정
  - 얼굴 등록 상태를 확인하고 얼굴 정보를 등록하거나 해제할 수 있습니다.
- 찜한 상품
  - 내가 찜한 상품을 모아서 보고, 필요하면 찜을 해제할 수 있습니다.
- 내가 평가한 상품
  - 내가 남긴 별점, 자극 여부, 재구매 의사 평가를 확인할 수 있습니다.
- 최근 본 상품
  - 최근에 상세 페이지에서 봤던 상품을 다시 확인할 수 있습니다.
- 내가 남긴 의견
  - 내가 작성한 댓글을 모아서 보고 수정하거나 삭제할 수 있습니다.
- 추천 피드백 관리
  - 추천에서 숨긴 상품이나 추천 피드백 내역을 확인하고 취소할 수 있습니다.


### 3.5 얼굴 로그인 기능

얼굴 로그인은 Python FastAPI 서버와 연동하여 구현했습니다.

카메라로 촬영한 얼굴 이미지를 Python 서버로 전달하면, InsightFace에서 buffalo_l 모델을 사용해 얼굴을 찾고 얼굴 특징 벡터를 추출합니다. 등록 시에는 여러 각도의 얼굴 이미지를 사용해서 대표 얼굴 벡터를 만들고, 로그인 시에는 현재 얼굴 벡터와 저장된 벡터의 유사도를 비교합니다.

얼굴만으로 사용자를 확실히 구분하기 어려운 경우에는 추가 각도 촬영을 요구하거나 비밀번호 로그인을 안내하도록 했습니다.

### 3.6 관리자 기능

관리자는 서비스 운영에 필요한 기능을 사용할 수 있습니다.
주요 기능은 다음과 같습니다.

- 관리자 대시보드
  - 전체 상품 수, 숨김 상품 수, 신고 대기 수 같은 운영 현황을 한눈에 볼 수 있습니다.
- 상품 관리
  - 상품을 검색하고 플랫폼, 이미지 여부, 노출 상태 등을 기준으로 관리할 수 있습니다.
- 상품 숨김/복구
  - 문제가 있는 상품을 사용자 화면에서 숨기고, 필요하면 다시 복구할 수 있습니다.
- 추천 제외/포함
  - 상품은 그대로 두고 추천 결과에서만 제외하거나 다시 포함할 수 있습니다.
- 상품 품질 상태 관리
  - 이미지 없음, 리뷰 부족, 주의 신호 높음 같은 상태를 관리자가 확인할 수 있습니다.
- 운영 메모 관리
  - 관리자가 상품별로 확인한 내용이나 처리 이유를 메모로 남길 수 있습니다.
- 회원 댓글 관리
  - 회원이 작성한 댓글을 확인하고 부적절한 댓글을 삭제하거나 복구할 수 있습니다.
- 신고 댓글 처리
  - 사용자가 신고한 댓글을 확인하고 관리자가 직접 처리할 수 있습니다.
- 관리자 활동 로그 확인
  - 상품 숨김, 추천 제외, 신고 처리 같은 관리자 작업 이력을 확인할 수 있습니다.

상품을 숨기거나 추천에서 제외할 때 원본 상품 데이터를 삭제하지 않고, 별도의 운영 상태 테이블에서 관리하도록 설계했습니다. 이렇게 하면 원본 크롤링 데이터를 보존하면서도 서비스 운영을 할 수 있습니다.

## 4. 데이터 설계 방향

BeautyLens는 데이터의 성격에 따라 테이블을 분리했습니다.

크게 다음과 같은 데이터로 나누었습니다.

- 회원 데이터
  - 로그인 정보, 닉네임, 피부 타입, 피부 관심사, 권한 정보를 저장합니다.
- 상품 데이터
  - 크롤링한 상품명, 브랜드, 플랫폼, 이미지, 상품 URL, 추천 점수 등을 저장합니다.
- 크롤링 리뷰 데이터
  - 외부 플랫폼에서 수집한 리뷰를 상품과 연결해서 저장합니다.
- 회원 댓글 데이터
  - BeautyLens 회원이 직접 작성한 자유 의견을 저장합니다.
- 상품 평가 데이터
  - 별점, 자극 여부, 재구매 의사처럼 추천 품질에 활용할 수 있는 평가를 저장합니다.
- 찜 데이터
  - 사용자가 관심 상품으로 저장한 상품 정보를 관리합니다.
- 최근 본 상품 및 사용자 행동 데이터
  - 상품 조회, 상세 조회, 찜, 평가 같은 사용자 행동을 기록합니다.
- 추천 피드백 데이터
  - 추천 결과에 대한 좋아요, 별로예요, 관심 없음 같은 반응을 저장합니다.
- 얼굴 인증 데이터
  - 얼굴 로그인에 필요한 얼굴 특징값과 등록 상태를 회원 정보와 분리해서 저장합니다.
- 관리자 운영 상태 데이터
  - 상품 숨김, 추천 제외, 품질 상태, 운영 메모를 원본 상품과 분리해서 관리합니다.
- 관리자 활동 로그 데이터
  - 관리자가 어떤 작업을 했는지 나중에 확인할 수 있도록 작업 이력을 저장합니다.

크롤링 리뷰와 회원 댓글은 출처가 다르기 때문에 분리했습니다. 또한 별점, 자극 여부, 재구매 의사 같은 평가는 자유 댓글과 목적이 다르기 때문에 별도 테이블로 관리했습니다.
관리자 기능에서도 상품을 직접 삭제하지 않고 숨김 상태나 추천 제외 상태를 별도 테이블에 저장하여 원본 데이터를 보존하도록 했습니다.

## 5. 사용 기술

프로젝트는 다음 기술을 사용했습니다.

- Spring MVC 6.x
  - 웹 요청을 받고, 서비스 로직을 실행하고, API 응답을 보내는 데 사용했습니다.
- JSP
  - 사용자 화면과 관리자 화면을 만드는 데 사용했습니다.
- CSS
  - 상품 카드, 마이페이지, 관리자 화면 스타일을 구성하는 데 사용했습니다.
- Vanilla JavaScript
  - 찜, 평가, 댓글, 얼굴 로그인 같은 화면 동작과 API 호출을 처리하는 데 사용했습니다.
- Oracle XE
  - 회원, 상품, 리뷰, 댓글, 평가, 관리자 데이터를 저장하는 DB로 사용했습니다.
- MyBatis XML
  - SQL을 직접 작성하고 Java 코드와 DB를 연결하는 데 사용했습니다.
- Maven WAR
  - 프로젝트를 빌드하고 Tomcat에 올릴 WAR 파일을 만드는 데 사용했습니다.
- Tomcat 11
  - Spring MVC 웹 애플리케이션을 실행하는 서버로 사용했습니다.
- Python FastAPI
  - 얼굴 인식 기능을 Spring 서버와 분리해서 별도 서버로 구현하는 데 사용했습니다.
- InsightFace
  - 얼굴을 찾고 얼굴 특징 벡터를 추출하는 데 사용했습니다.
- ONNX Runtime
  - InsightFace 모델을 실행하는 데 사용했습니다.
- OpenCV fallback
  - InsightFace가 동작하지 않을 때를 대비한 보조 얼굴 검출 방식으로 사용했습니다.

프론트엔드는 JSP와 JavaScript 기반으로 구현했습니다. 얼굴 인증은 Spring 서버와 Python 서버를 분리하여, Spring은 회원, 세션, DB 관리를 담당하고 Python은 얼굴 인식 처리를 담당하도록 구성했습니다.

## 6. 보완할 점, 부족한 점

주요 기능 구현에 집중하다 보니 일부 기능은 제한적으로 구현되었습니다.

- 회원가입은 단순하게 구현했습니다.
  - 아이디, 비밀번호, 닉네임 정도만 입력하면 가입할 수 있게 만들었습니다. 실제 서비스라면 비밀번호 확인, 아이디 중복 확인, 약관 동의 같은 과정이 더 필요합니다.
- 브랜드, 가격대, 카테고리 같은 세부 검색 필터는 아직 부족합니다.
  - 생각은 했지만 시간상 충분히 구현하지 못했습니다. 현재는 상품명, 플랫폼, 피부 타입, 이미지 여부 중심의 기본 필터만 들어가 있습니다.
- 프론트엔드를 React로 분리해서 구현해 보고 싶었지만 못했습니다.
  - 이번에는 Spring MVC와 JSP 기반 기능을 먼저 완성하는 데 집중해서 JSP와 JavaScript 구조를 유지했습니다.
- 상품 이미지를 아직 다 수집하지 못했습니다.
  - 전체 상품 이미지가 모두 들어간 것은 아니고, 시연에 필요한 약 200개 정도의 이미지만 우선 넣었습니다.
- 크롤링한 데이터에 의존을 해서 시간이 지났을 때 새로운 인기상품의 확장을 생각하지 않았습니다.
  - 크롤링을 주기적으로 추가하거나 이 사이트에 다른 크롤링 데이터를 넣어서 그 데이터에 맞게 사이트를 변환할 수 있게 보다 이식성이 좋게 설계를 해보는 방향도 있을 것 같습니다.

## 7. 결론

BeautyLens는 크롤링한 화장품 상품과 리뷰 데이터를 기반으로 피부 타입별 추천을 제공하고, 사용자의 평가와 댓글, 관리자의 운영 기능을 결합한 Spring MVC 기반 화장품 추천 웹 서비스입니다.
본 프로젝트를 통해 데이터 수집, DB 설계, 사용자 기능, 관리자 기능, 얼굴 인증, 추천 화면 구성까지 웹 서비스 전반의 흐름을 구현해 볼 수 있었습니다. 사이트 내 사용자의 평가가 있지만 현재로서는 웹크롤링한 리뷰 데이터가 주가 맞고, 이 흐름은 많이 정적이라고 생각하고 있습니다.

하지만 그냥 크롤링한 데이터로 CRUD가 들어가기만 하면 되겠지 라는 생각으로 조금 더 서비스를 생각하지 못했습니다.
