# 발표 시연 대본

## 시연 준비 계정

| 역할 | 계정 |
|---|---|
| 사용자 | `test01 / 1234` |
| 관리자 | `admin / 1234` |

기본 주소:

```text
http://localhost:8088/beautylens-mvc/
```

## 1. 프로젝트 소개

- 이동 URL: `/beautylens-mvc/`
- 보여줄 화면: 홈 화면
- 발표 멘트:
  - "BeautyLens는 크롤링한 화장품 상품과 리뷰 데이터를 기반으로, 피부 타입에 맞는 상품을 추천하는 웹 서비스입니다."
  - "단순히 상품과 리뷰만 보여주는 것이 아니라, 사용자의 찜, 평가, 댓글, 최근 본 상품 같은 활동 데이터와 관리자 운영 기능까지 붙여 실제 서비스 흐름처럼 구성했습니다."
- 실패했을 때 대체 멘트:
  - "홈 화면 접속이 늦으면 바로 상품 목록으로 이동해서 데이터와 기능 구조를 먼저 보여드리겠습니다."

## 2. ERD 설명

- 이동 파일: `docs/final/02_database_and_erd.md`
- 보여줄 화면: ERD 이미지
- 발표 멘트:
  - "상품과 크롤링 리뷰는 `BL_PRODUCTS`, `BL_REVIEWS`로 분리했습니다."
  - "회원이 남기는 댓글은 외부 리뷰와 성격이 다르기 때문에 `BL_PRODUCT_COMMENTS`로 따로 두었습니다."
  - "관리자가 상품을 숨길 때 원본 상품을 삭제하지 않고 `BL_PRODUCT_ADMIN_FLAGS`에서 운영 상태만 관리하도록 설계했습니다."
- 실패했을 때 대체 멘트:
  - "이미지가 안 보이면 Mermaid ERD와 테이블 설명을 기준으로 설명하겠습니다."

## 3. 메인 화면

- 이동 URL: `/beautylens-mvc/`
- 클릭할 버튼: `추천 받기`, `상품 둘러보기`
- 보여줄 화면: 서비스 소개, 추천/상품 CTA
- 발표 멘트:
  - "홈에서는 기술 설명보다 사용자가 바로 상품 탐색이나 추천으로 이동할 수 있게 구성했습니다."
- 실패했을 때 대체 멘트:
  - "홈은 진입 화면이므로, 핵심 기능인 상품 목록과 추천 페이지를 직접 보여드리겠습니다."

## 4. 상품 목록

- 이동 URL: `/beautylens-mvc/products`
- 클릭할 버튼: 검색/정렬, 상품 카드 `자세히 보기`, `찜하기`
- 보여줄 화면: 상품 카드 목록
- 발표 멘트:
  - "상품 카드에서는 너무 많은 버튼을 노출하지 않고, 자세히 보기와 찜하기 중심으로 정리했습니다."
  - "숨김 처리된 상품은 일반 상품 목록에서 제외됩니다."
- 실패했을 때 대체 멘트:
  - "목록 API는 `/api/products`로 확인할 수 있고, Mapper에서 숨김 상품 제외 조건을 적용했습니다."

## 5. 피부 타입 추천

- 이동 URL: `/beautylens-mvc/recommend`
- 클릭할 버튼: 피부 타입 버튼, 플랫폼 필터, 추천 카드 자세히 보기
- 보여줄 화면: 추천 결과 카드
- 발표 멘트:
  - "추천은 크롤링 리뷰 기반 추천 점수를 기본으로 하고, 내부 별점, 찜, 조회, 댓글, 추천 피드백을 보조 점수로 반영했습니다."
  - "관리자가 추천 제외한 상품과 사용자가 관심 없음 처리한 상품은 추천 결과에서 제외됩니다."
- 실패했을 때 대체 멘트:
  - "추천 API는 `/api/recommendations?skinType=dry&size=20`에서 확인할 수 있습니다."

## 6. 상품 상세

- 이동 URL: `/beautylens-mvc/products/1446` 또는 존재하는 상품 ID
- 클릭할 버튼: 외부 상품 페이지, 찜하기
- 보여줄 화면: 상품 요약, 추천 근거, 외부 리뷰 분석, 사이트 사용자 평가
- 발표 멘트:
  - "상세 화면은 외부 리뷰 분석, 사이트 사용자 평가, 회원 의견을 섞지 않고 분리했습니다."
  - "외부 리뷰 분석은 크롤링 데이터이고, 사이트 사용자 평가는 BeautyLens 안에서 쌓이는 내부 데이터입니다."
- 실패했을 때 대체 멘트:
  - "상품 ID가 숨김 처리되어 404가 나면 다른 상품 상세로 이동해 설명하겠습니다."

## 7. 찜/평가/댓글

- 로그인 계정: `test01 / 1234`
- 이동 URL: `/beautylens-mvc/products/1446`
- 클릭할 버튼: `찜하기`, `평가 저장`, `내 평가 삭제`, `의견 작성`, 댓글 `수정`, `삭제`, `신고`
- 보여줄 화면: 상품 상세 하단 평가/댓글 영역
- 발표 멘트:
  - "평가는 별점, 자극 여부, 재구매 의사처럼 추천 품질에 쓸 수 있는 정량 데이터입니다."
  - "댓글은 다른 회원에게 보이는 자유 의견이라 평가와 분리했습니다."
- 실패했을 때 대체 멘트:
  - "로그인이 풀렸다면 다시 로그인 후 같은 상품 상세로 돌아와 시연하겠습니다."

## 8. 마이페이지

- 이동 URL: `/beautylens-mvc/mypage`
- 클릭할 메뉴: 내 프로필, 피부 타입, 얼굴 로그인, 찜한 상품, 내가 평가한 상품, 최근 본 상품, 내가 남긴 의견, 추천 피드백
- 보여줄 화면: 사이드바/탭 구조
- 발표 멘트:
  - "마이페이지는 긴 세로 목록이 아니라 계정 대시보드처럼 탭으로 나누었습니다."
  - "사용자는 자신이 남긴 찜, 평가, 댓글, 최근 본 상품을 다시 확인하고 관리할 수 있습니다."
- 실패했을 때 대체 멘트:
  - "마이페이지 API는 `/api/members/me/favorites`, `/api/members/me/ratings`, `/api/members/me/recent-products`로 나뉘어 있습니다."

## 9. 얼굴 등록/얼굴 로그인

- Python 서버: `http://127.0.0.1:8090/health`
- 이동 URL: `/beautylens-mvc/mypage`, `/beautylens-mvc/login`
- 클릭할 버튼: `얼굴 등록`, `얼굴로 로그인`
- 보여줄 화면: 단계형 얼굴 등록, 자동 얼굴 로그인 모달
- 발표 멘트:
  - "얼굴 등록은 정면, 왼쪽, 오른쪽, 위쪽, 아래쪽 이미지를 등록합니다."
  - "얼굴 로그인은 정면으로 자동 확인하고, 애매하면 추가 각도를 요청하도록 보수적으로 처리했습니다."
- 실패했을 때 대체 멘트:
  - "얼굴 서버나 카메라 권한 문제가 있으면 비밀번호 로그인은 그대로 유지되도록 설계했습니다."

## 10. 관리자 대시보드

- 로그인 계정: `admin / 1234`
- 이동 URL: `/beautylens-mvc/admin`
- 보여줄 화면: 통계 카드와 운영 요약
- 발표 멘트:
  - "관리자는 상품 수, 리뷰 수, 댓글 수, 신고 대기 수, 내부 평가 수, 얼굴 등록 회원 수를 한눈에 확인할 수 있습니다."
- 실패했을 때 대체 멘트:
  - "관리자 API는 `/api/admin/summary`이며, 일반 사용자는 접근할 수 없습니다."

## 11. 관리자 상품 관리

- 이동 URL: `/beautylens-mvc/admin/products`
- 클릭할 버튼: 검색, 숨김/복구, 추천 제외/포함, 품질 상태, 메모 저장
- 보여줄 화면: 상품 운영 테이블
- 발표 멘트:
  - "관리자는 원본 상품 데이터를 삭제하지 않고 운영 상태만 별도 테이블로 관리합니다."
  - "숨김은 사용자 화면에서 제외하고, 추천 제외는 추천 결과에서만 제외합니다."
- 실패했을 때 대체 멘트:
  - "실제 상품을 잘못 숨기지 않도록 시연용 상품으로만 테스트하겠습니다."

## 12. 관리자 댓글/신고 관리

- 이동 URL: `/beautylens-mvc/admin/comments`, `/beautylens-mvc/admin/comment-reports`
- 클릭할 버튼: 댓글 삭제/복구, 신고 처리
- 보여줄 화면: 댓글 목록, 신고 목록
- 발표 멘트:
  - "회원 댓글은 soft delete 방식으로 삭제 상태를 관리하고, 신고는 별도 테이블에서 처리 상태를 관리합니다."
- 실패했을 때 대체 멘트:
  - "신고 데이터가 없으면 신고된 댓글이 없다는 빈 상태를 보여드리겠습니다."
